# 数据库体检报告脚本使用说明

## 📋 功能说明

`database_health_check.py` 脚本用于生成数据库的详细体检报告，包括：

1. **章节统计**：统计每个章节（1-25）的题目数量
2. **状态评估**：根据题目数量评估章节状态（充足/一般/预警）
3. **匹配验证**：检查未分配和无效的章节编号
4. **分布分析**：分析章节题目分布情况
5. **健康评分**：综合评估数据库健康状态

## 🚀 使用方法

### 基本使用

```bash
cd /path/to/assets
python3 maintenance/scripts/database_health_check.py
```

### 保存报告到文件

```bash
python3 maintenance/scripts/database_health_check.py > maintenance/reports/database_health_report.txt
```

## 📊 报告内容

### 1. 章节题目数量统计

显示每个章节（1-25）的题目数量和状态：

| 章节编号 | 题目数量 | 状态 |
|---------|---------|------|
| 1       | 727     | ✅ 充足 |
| 2       | 583     | ✅ 充足 |
| ...     | ...     | ... |

**状态说明**：
- ✅ **充足**：题目数量 ≥ 100
- ⚠️ **一般**：题目数量 50-99
- 🔴 **预警**：题目数量 < 50

### 2. 总体统计

- 题目总数
- 已分配章节的题目数（1-25）
- 未分配题目数（chapter IS NULL）
- 无效章节编号题目数（不在 1-25 范围内）
- 章节分配率

### 3. 匹配验证

检查以下问题：
- 未分配章节的题目（chapter IS NULL）
- 无效章节编号的题目（不在 1-25 范围内）

### 4. 章节覆盖情况

检查是否有章节（1-25）完全没有题目。

### 5. 章节分布分析

分析题目分布情况：
- 最多题目的章节
- 最少题目的章节
- 平均题目数量

### 6. 数据库健康评分

综合评估数据库健康状态（0-100分）：
- 🌟 **优秀** (90-100分)：数据库状态良好
- ✅ **良好** (75-89分)：存在少量问题
- ⚠️ **一般** (60-74分)：存在一些问题
- 🔴 **需要改进** (<60分)：存在严重问题

## 🔍 状态评估标准

### 章节状态

- **充足** (≥ 100题)：章节题目数量充足，可以正常使用
- **一般** (50-99题)：章节题目数量一般，建议补充
- **预警** (< 50题)：章节题目数量不足，需要关注

### 健康评分规则

基础分数：100分

扣分项：
- 未分配题目：根据比例扣分（最多20分）
- 无效章节编号：直接扣30分
- 缺失章节：每个缺失章节扣2分
- 题目数量过少：每个预警章节扣1分

评分等级：
- 🌟 优秀 (90-100分)
- ✅ 良好 (75-89分)
- ⚠️ 一般 (60-74分)
- 🔴 需要改进 (<60分)

## ⚙️ 配置说明

### 数据库路径

默认数据库路径：`assets/italy_quiz.db`

如需修改，在脚本中修改 `DB_PATH` 变量：

```python
DB_PATH = 'your/path/to/italy_quiz.db'
```

### 状态阈值

如需修改状态评估阈值，在 `get_status` 函数中修改：

```python
def get_status(count: int) -> str:
    if count >= 100:  # 修改这个值
        return '✅ 充足'
    elif count >= 50:  # 修改这个值
        return '⚠️  一般'
    else:
        return '🔴 预警'
```

## 🐛 故障排除

### 问题1：数据库文件不存在

**错误信息**：`❌ 数据库文件不存在: assets/italy_quiz.db`

**解决方法**：
1. 确认数据库文件路径正确
2. 确认文件确实存在
3. 检查文件权限

### 问题2：数据库连接失败

**错误信息**：`❌ 数据库连接失败: ...`

**解决方法**：
1. 检查数据库文件是否损坏
2. 确认文件权限可读
3. 尝试用 SQLite 工具打开数据库文件

## 📝 使用建议

1. **定期检查**：建议在每次数据更新后运行此脚本
2. **保存报告**：将报告保存到文件，便于追踪数据库状态变化
3. **关注预警**：对于标记为"预警"的章节，考虑补充题目
4. **处理未分配题目**：对于未分配的题目，可以考虑分配章节或移除
