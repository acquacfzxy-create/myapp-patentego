# 数据库章节重新分配脚本使用说明

## 📋 脚本功能

`reassign_chapters.py` 脚本用于根据原始 JSON 文件中的嵌套结构，将 JSON 的顶级键（章节名）映射到数据库的 `chapter` 字段（1-25）。

## 🎯 核心特性

1. **自动映射**：根据 JSON 文件的顶级键顺序，自动创建 1-25 的章节映射
2. **智能匹配**：使用文本相似度算法（`difflib.SequenceMatcher`）在数据库中匹配题目
3. **高准确率**：先精确匹配，再相似度匹配（阈值 0.85），确保匹配准确性
4. **完整验证**：处理完成后自动验证分配结果，显示各章节题目数量统计

## 🚀 使用方法

### 前置条件

1. **JSON 文件路径**：脚本会在以下路径查找 JSON 文件：
   - `./quizPatenteB2023.json`
   - `../patente/quizPatenteB2023.json`

2. **数据库文件**：`assets/italy_quiz.db` 必须存在

3. **Python 环境**：Python 3.6+（使用标准库，无需额外依赖）

### 运行脚本

```bash
cd /path/to/assets
python3 maintenance/scripts/reassign_chapters.py
```

### 执行步骤

脚本会自动执行以下步骤：

1. ✅ **查找 JSON 文件**：自动在预设路径中查找 JSON 文件
2. ✅ **读取 JSON 数据**：解析 JSON 文件的嵌套结构
3. ✅ **构建章节映射**：根据 JSON 文件的顶级键顺序创建 1-25 的映射
4. ✅ **连接数据库**：连接到 `assets/italy_quiz.db`
5. ✅ **检查字段**：如果 `chapter` 字段不存在，自动添加
6. ✅ **处理章节**：遍历每个章节，提取题目并匹配数据库记录
7. ✅ **更新数据库**：将匹配的题目的 `chapter` 字段更新为对应的章节编号
8. ✅ **验证结果**：统计各章节题目数量，显示分配结果

## 🔍 技术细节

### 章节映射逻辑

脚本会自动从 JSON 文件的顶级键（按顺序）创建映射：

```python
# JSON 文件的顶级键顺序就是章节顺序
keys = ['definizioni-generali-doveri-strada', 'segnali-pericolo', ...]
# 映射为：
mapping = {
    'definizioni-generali-doveri-strada': 1,
    'segnali-pericolo': 2,
    ...
}
```

### 题目匹配算法

1. **文本标准化**：
   - 转换为小写
   - 去除首尾空格
   - 标准化空白字符

2. **匹配策略**：
   - **精确匹配**：标准化后的文本完全一致
   - **相似度匹配**：使用 `SequenceMatcher` 计算相似度（阈值 0.85）

3. **性能优化**：
   - 先精确匹配，命中即返回
   - 精确匹配失败才进行相似度计算

### JSON 结构解析

JSON 文件结构：
```json
{
  "章节名1": {
    "子分类1": [
      {"q": "题目内容", "a": true/false, "img": "图片路径"},
      ...
    ],
    "子分类2": [...]
  },
  "章节名2": {...}
}
```

脚本会递归遍历所有子分类，提取所有题目。

## ⚠️ 注意事项

1. **备份数据库**：运行脚本前建议备份数据库：
   ```bash
   cp assets/italy_quiz.db assets/italy_quiz.db.backup
   ```

2. **未分配的题目**：如果数据库中有 JSON 文件不包含的题目，这些题目会保持 `chapter = NULL`。

3. **执行时间**：处理约 7000 道题目需要 1-2 分钟，请耐心等待。

4. **匹配准确率**：脚本使用相似度阈值 0.85，如果匹配率低于预期，可以调整阈值（在代码中修改 `threshold` 参数）。

## 🔧 自定义配置

### 修改 JSON 文件路径

在脚本中修改 `JSON_FILE_PATHS` 列表：

```python
JSON_FILE_PATHS = [
    '/your/custom/path/quizPatenteB2023.json',
    './your_file.json',
]
```

### 修改数据库路径

在脚本中修改 `DB_PATH`：

```python
DB_PATH = 'your/path/to/italy_quiz.db'
```

### 调整相似度阈值

在 `find_question_by_content` 函数调用时修改 `threshold` 参数：

```python
question_id = find_question_by_content(db, question_text, lang='it', threshold=0.90)
```

- 阈值越高，匹配越严格（可能匹配率降低）
- 阈值越低，匹配越宽松（可能误匹配增加）
- 推荐值：0.85（默认）

## 🐛 故障排除

### 问题1：找不到 JSON 文件

**错误信息**：`❌ 未找到 JSON 文件！`

**解决方法**：
1. 确认 JSON 文件路径是否正确
2. 在脚本中修改 `JSON_FILE_PATHS` 列表，添加正确的路径

### 问题2：数据库连接失败

**错误信息**：`❌ 数据库连接失败: ...`

**解决方法**：
1. 确认数据库文件 `assets/italy_quiz.db` 存在
2. 确认文件权限可读可写
3. 检查数据库文件是否损坏

### 问题3：匹配率过低

**现象**：匹配率远低于 100%

**解决方法**：
1. 检查 JSON 文件和数据库中的题目文本是否一致
2. 尝试降低相似度阈值（如 0.80）
3. 检查数据库中的题目是否包含 JSON 文件中的所有题目
